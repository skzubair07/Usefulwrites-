Database Schema

  • wp_pade_queue: post_id, payload_json, status, retry_count, created_at, updated_at.
  • wp_pade_logs: timestamp, platform, post_id, http_code, success_flag, raw_response.

Queue Lifecycle

pending → approved → scheduled → processing → completed / failed

Data Flow Diagram (Text)

Post Publish -> Queue Insert
Queue Cron/Event -> Queue Processor
Queue Processor -> AI Engine (caption)
Queue Processor -> Router -> Platform Class
Platform Class -> Logger::log()
Logger -> wp_pade_logs

How to Add a New Platform

  1. Create a new class in platforms/ with a send(array $payload): array method.
  2. Inject PADE_Settings and PADE_Logger.
  3. Call $logger->log() after the request.
  4. Register it in PADE_Router::dispatch() and PADE_Router::platforms().

How AI Integration Works

PADE_AI_Engine builds a platform prompt using placeholders: {post_title}, {post_content}, {platform_name}. It sends the prompt to OpenAI Chat Completions using admin-defined model and max tokens. All raw AI responses are logged in wp_pade_logs under platform ai-engine.
